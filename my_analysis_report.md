# SSD故障日志深度分析与预测建模报告

## 1. 数据集说明

- 数据来源：阿里云公开数据集 `dcbrain/ssd_open_data`
- 使用数据：
  - `smart_log_20191231.csv`：2019-12-31 单日 SMART 日志
  - `ssd_failure_tag.csv`：SSD 故障标签
  - `location_info_of_ssd.csv`：应用、机架、节点、槽位等部署位置信息
- 样本粒度：以 `(model, disk_id)` 作为一条 SSD 样本
- 标签含义：`failure=0` 表示健康盘，`failure=1` 表示故障盘

## 2. 数据处理与特征构建

- 读取 SMART 日志中的关键字段：`r_5`, `r_9`, `r_12`, `r_175`, `n_5`, `n_9`, `n_12`, `n_175`
- 按 `(model, disk_id)` 合并故障标签，未匹配到故障标签的样本记为健康盘
- 合并 `location_info` 中的 `app`, `rack_id`, `node_id`, `slot_id`
- 对 `slot_id` 缺失值标记为 `missing`，样本数过少的槽位合并为 `rare_slot`
- 建模时保留 `model`, `app`, `slot_id_grouped`，不直接使用高基数的 `rack_id` 和 `node_id`

处理后样本分布：

| 指标 | 数值 |
|---|---:|
| SMART 样本数 | 706,182 |
| 故障样本数 | 3,459 |
| 健康样本数 | 702,723 |
| 故障占比 | 0.49% |
| 健康/故障比例 | 203.16:1 |
| `location_info` 匹配率 | 100.00% |
| `slot_id` 缺失率 | 18.02% |

## 5. 故障预测建模

### 5.1 建模目标与特征

目标：基于单日 SMART 指标、型号和部分位置信息，对 SSD 是否为故障盘进行风险排序。

使用三组模型进行对比：

- Baseline：仅使用 SMART 数值特征
- Model-aware：SMART 数值特征 + `model`
- Location-aware：SMART 数值特征 + `model` + `app` + `slot_id_grouped`

建模处理：

- 缺失数值使用中位数填充
- 数值特征标准化
- 类别特征独热编码
- 使用逻辑回归模型
- 设置 `class_weight='balanced'` 缓解极端类别不平衡
- 使用 ROC-AUC、Average Precision、故障召回率、Top-K 命中率评估模型

### 5.2 模型对比结果

| 指标 | Baseline：SMART | Model-aware：SMART + model | Location-aware：SMART + model + app + slot |
|---|---:|---:|---:|
| 训练集样本数 | 564,945 | 564,945 | 564,945 |
| 测试集样本数 | 141,237 | 141,237 | 141,237 |
| Accuracy | 0.6908 | 0.7206 | 0.8065 |
| Failed Precision | 0.0130 | 0.0139 | 0.0197 |
| Failed Recall | 0.8280 | 0.8020 | 0.7890 |
| Failed F1 | 0.0256 | 0.0274 | 0.0384 |
| ROC-AUC | 0.7993 | 0.8208 | 0.8495 |
| Average Precision | 0.0214 | 0.0385 | 0.0385 |

- 加入 `model` 后，ROC-AUC 从 0.7993 提升到 0.8208。
- 继续加入 `app` 和 `slot_id_grouped` 后，ROC-AUC 提升到 0.8495。
- Location-aware 模型的故障类精确率提升到 1.97%，误报情况有所改善。
- 测试集故障率只有约 0.49%，随机排序的 Average Precision 约为 0.0049；Location-aware 模型的 Average Precision 为 0.0385，约为随机水平的 7.9 倍。

### 5.3 Location-aware 模型混淆矩阵

|  | 预测健康 | 预测故障 |
|---|---:|---:|
| 实际健康 | 113,366 | 27,179 |
| 实际故障 | 146 | 546 |

- 模型召回了 78.90% 的故障样本。
- 由于故障样本极少，仍然存在较多误报。
- 该模型更适合作为高风险磁盘排序工具，而不是直接作为自动告警系统。

## 6. 结论与建议

- SSD 故障样本极少，健康/故障比例约为 203.16:1，因此不能只看 Accuracy，应重点关注 Recall、Precision、ROC-AUC、Average Precision 和 Top-K 命中率。
- `r_5` 是最明显的故障相关 SMART 指标，故障盘中的均值显著高于健康盘。
- `r_12` 和 `r_175` 包含一定辅助信息，但 `r_175` 缺失率较高，不适合单独依赖。
- 型号差异明显，A3 是故障率最高的型号，应重点监控。
- DAE、NAS 应用故障率明显高于整体水平，说明业务类型与 SSD 故障风险有关。
- 部分机架存在明显故障聚集，建议在运维中关注高风险机架的环境和批次因素。
- 加入 `model`, `app`, `slot_id` 后，模型 ROC-AUC 提升到 0.8495，说明业务和位置特征能提升风险排序能力。
- 当前数据只有单日 SMART 日志，模型适合作为风险排序 baseline；若要进一步提升预测能力，建议加入连续多日 SMART 时间序列、更多环境信息，并尝试树模型或不平衡学习方法。
